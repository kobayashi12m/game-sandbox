● 段階的実装計画

フェーズ 1: 基本描画システム（最も簡単）

クライアント:

- Canvas セットアップ
- 円（コア）と外枠（ガード）の描画
- 固定位置に 1 つのプレイヤーを表示

サーバー:

- プレイヤーの初期位置データ構造定義
- WebSocket 接続時にプレイヤー ID と初期位置を送信

フェーズ 2: プレイヤー移動

クライアント:

- マウス追従でプレイヤー移動
- 移動データをサーバーに送信

サーバー:

- 移動データの受信と保存
- 他のクライアントへの配信（まだ 1 人でも）

フェーズ 3: マルチプレイヤー表示

クライアント:

- 複数プレイヤーの描画
- 他プレイヤーの位置更新

サーバー:

- プレイヤーリスト管理
- 全プレイヤーへの状態配信

フェーズ 4: 基本的な衝突判定

クライアント:

- 円形の衝突判定（コアとガード）
- 衝突時のビジュアルフィードバック

サーバー:

- 衝突判定の検証
- ゲーム結果の処理

フェーズ 5: ガード変形（中難易度）

クライアント:

- ガードの基本変形（拡大・縮小）
- マウスドラッグでの形状変更

サーバー:

- ガード形状データの同期

フェーズ 6: ゲームロジック完成

- スコアシステム
- プレイヤーの成長
- リスポーン処理

フェーズ 7: Redis 統合（高難易度）

- ゲーム状態の永続化
- リーダーボード実装

フェーズ 8: UI・仕上げ

- メニュー画面
- 設定機能
- エフェクト追加

2. **プレイヤー描画**

   ```javascript
   // 単純な円形で表現
   - コア: 小さい円（赤色）
   - ガード: 大きい円（青色、半透明）

   ```

3. **WebSocket 接続維持**
   ```javascript
   // 既存のチャット機能を改修

   ```

#### サーバー側

1. **プレイヤー構造体**

   ```go
   type Player struct {
       ID       string

       Y        float64
       CoreSize float64
       GuardSize float64

   ```

2. **初期化処理**

   ```go
   // 新規接続時
   - プレイヤーID生成
   - 初期位置設定 (400, 300)
   - 初期サイズ設定
   ```

3. **メッセージ送信**
   ```go
   // WebSocket接続確立時
   - プレイヤー情報をJSON形式で送信
   ```

### 成功基準

- ブラウザを開くと Canvas が表示される
- 中央に赤い円（コア）と青い円（ガード）が表示される
- コンソールにプレイヤー ID が表示される

### 予想作業時間

- クライアント: 30 分
- サーバー: 20 分
- デバッグ: 10 分
