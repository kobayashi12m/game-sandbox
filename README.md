# コア・ガード.io - ゲームコンセプト

## 基本概念

プレイヤーは中心にある **コア** とそれを守る **ガード（外殻）** を操作する新感覚の io ゲームです。

## ゲーム要素

### コア

- プレイヤーの中心部分
- **触れられたら即座に敗北**
- 移動は可能だが、常にガードに覆われている必要がある

### ガード（外殻）

- コアを包囲する防御層
- **自在に伸縮・変形可能**
- サイズを大きくすることができる唯一の要素
- 他プレイヤーのコアに触れることで勝利

## 勝利条件

**勝利**: 自分のガードで相手のコアに触れる
**敗北**: 自分のコアが相手のガードに触れられる

## 戦略要素

### 攻撃戦略

- ガードを伸ばして相手のコアを狙う
- 相手の隙をついてコアに接触する
- ガードのサイズを活かした包囲攻撃

### 防御戦略

- コアを完全に覆い隠す
- ガードの形状を工夫して死角を作らない
- 相手の攻撃を予測した回避行動

### 成長システム

- 他プレイヤーを倒すことでガードサイズが拡大
- より大きなガードで複数の敵を同時に攻撃可能
- サイズが大きくなるほどコアの防御も困難に

## ゲームプレイの特徴

- **攻守一体**: ガードは攻撃と防御の両方を担う
- **リスク・リワード**: 攻撃のために伸ばすとコアが露出するリスク
- **戦術的多様性**: ガードの形状変化による無限の戦術パターン
- **瞬間的勝負**: 一瞬の隙が勝敗を分ける緊張感

## 技術仕様

### コア技術スタック

#### ゲームエンジン & グラフィックス
- **PixiJS / Three.js**
  - ガードの滑らかな変形アニメーション実装
  - WebGLによる高性能レンダリング
  - パーティクルエフェクト（破壊時、成長時）
  - シェーダーによる視覚効果

#### リアルタイム通信
- **WebSocket (Gorilla WebSocket)**
  - 低遅延の双方向通信
  - バイナリプロトコルによる効率的なデータ転送
  - 自動再接続とハートビート機能

#### 状態管理 & 永続化
- **Redis**
  - **ゲームステート**: リアルタイムでのプレイヤー位置、ガード形状の同期
  - **マッチメイキング**: 待機中のプレイヤーキューと部屋管理
  - **リーダーボード**: Sorted Setによるランキング管理
  - **Pub/Sub**: 複数サーバー間でのイベント配信
  - **セッション**: プレイヤーの一時的な状態保存

#### バックエンド
- **Go**
  - 高並行性によるマルチプレイヤー対応
  - ゲームループの効率的な実行（60fps目標）
  - 物理演算と衝突判定処理

#### フロントエンド
- **React + TypeScript**
  - UIコンポーネント（メニュー、リーダーボード、設定）
  - 状態管理（Zustand/Jotai）
  - レスポンシブデザイン対応

### アーキテクチャ設計

```
┌─────────────┐     WebSocket      ┌──────────────┐     Redis Pub/Sub    ┌──────────────┐
│   Client    │ ←────────────────→ │  Game Server │ ←─────────────────→ │  Game Server │
│  (Browser)  │                     │   (Go #1)    │                      │   (Go #2)    │
└─────────────┘                     └──────────────┘                      └──────────────┘
      ↓                                     ↓                                      ↓
   PixiJS                                Redis                                  Redis
   Rendering                          ┌──────────┐                          (Replicated)
                                     │  Primary  │
                                     │   Redis   │
                                     └──────────┘
                                          ↓
                                    - Game Rooms
                                    - Player States
                                    - Leaderboards
                                    - Match Queue
```

### 主要機能の技術実装

#### 1. ガード変形システム
- **ベジェ曲線**による滑らかな形状制御
- **物理演算**による自然な動き
- **メッシュ変形**でリアルタイムアニメーション

#### 2. 衝突判定
- **SAT (Separating Axis Theorem)** による精密な判定
- **空間分割**（Quadtree）による最適化
- **予測的衝突検出**で遅延対策

#### 3. ネットワーク同期
- **状態補間**による滑らかな動き
- **クライアント予測**で操作遅延を軽減
- **デルタ圧縮**でデータ転送量削減

#### 4. スケーラビリティ
- **水平スケーリング**: Redis Pub/Subによるサーバー間通信
- **ルームベース**: プレイヤーを論理的に分離
- **ロードバランサー**: 接続の分散

### パフォーマンス目標
- レンダリング: 60fps維持
- ネットワーク遅延: 50ms以下（同一リージョン）
- 同時接続数: 1サーバーあたり1000プレイヤー
- ルームサイズ: 最大50プレイヤー/ルーム
